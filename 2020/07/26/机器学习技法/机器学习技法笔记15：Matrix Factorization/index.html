机器学习技法笔记15：Matrix Factorization

Posted on 2020-07-26 In 机器学习技法听课笔记 Valine:

0 说明

Matrix Factorization，也就是矩阵的分解，主要应用于推荐系统领域，用于实现隐含语义模型（Latent Factor Model）。通过矩阵分解，一方面可以减少运算量，另一方面可以很好的解决由于用户数目和物品数目过多引起的行为矩阵稀疏化问题。

这一讲从一个实际比赛的例子出发，讲解矩阵分解的用法。

1 Linear Network Hypothesis

Netflix 举办过一个比赛，这个比赛给参赛者 480189 个用户的总共 100480507 笔对 17770 部电影的评分，希望参赛者通过这些数据训练一个模型，这个模型可以尽可能准确地预测出用户对于尚未评分过的电影的评分。

用 (x~n=(n),yn=rnm)(\tilde{x}_n=(n),y_n=r_{nm})(x~n​=(n),yn​=rnm​) 表示用户 n 对电影 m 的评分，其中用户 id x~n\tilde{x}_nx~n​ 是一个类别特征，在这门课讲过的所有算法中能够直接使用类别特征的只有决策树和它的聚合模型，对于其他模型而言，使用类别特征需要先把它进行编码转成数值特征。这里我们使用最简单的一种编码方式：binary vector encoding，就是有几个类别就用一个几维向量表示这个分类变量，每个位置的值都是 0 或 1，比如对于血型变量我们可以用 A=(1,0,0,0)TA=(1,0,0,0)^TA=(1,0,0,0)T, B=(0,1,0,0)TB=(0,1,0,0)^TB=(0,1,0,0)T, AB=(0,0,1,0)TAB=(0,0,1,0)^TAB=(0,0,1,0)T, O=(0,0,0,1)TO=(0,0,0,1)^TO=(0,0,0,1)T 来表示。这种表示方法就是常见的 one-hot 编码。容易看出来，当类别数目过多时这样的表示向量会非常的稀疏。

现在我们希望建立一个 N−d~−MN-\tilde{d}-MN−d~−M 的网络，输入数据是使用 binary vector encoding 编码方式表示的用户 id，输出数据是用户对各个电影的评分向量 y=(rn1,…,rnM)y=(r_{n1},\ldots,r_{nM})y=(rn1​,…,rnM​)，未评分的电影的位置留空出来表示还未评分。

下图中展示了这个网络，中间层转换函数使用的线性的，这并没有什么特别的理由，只是因为此时输入向量是一个只有一个位置不是 0 的向量，所以中间的转换函数是什么对模型整体没有影响，只是对第一层的权重有影响，但是第一层转换后的结果应该都是一样的，所以直接使用线性函数方便计算，这样的模型我们称之为线性网络。

3 Stochastic Gradient Descent

这一节尝试使用随机梯度下降法优化这个问题，使用这个方法有几个原因，首先是常说的 SGD 具有的两条好的性质，那就是每一次迭代的效率更高并且更加容易操作/写代码，除此之外我们使用 SGD 还有一个特别的原因，那就是上一节的交替优化方法随着误差函数的变化有很大的差异，有时选择的误差函数可能导致交替优化在每一步都是很困难的，但是 SGD 不一样，我们不需要考虑损失函数的连加，只需要对单个损失函数求导，相对而言对于不同形式的误差函数优化起来都不会太困难。

以二次损失函数为例，我们的损失函数是：err(user n,movie m,rating rnm)=(rnm−wmTvn)2err(user\ n,movie\ m,rating\ r_{nm})=(r_{nm}-w^T_mv_n)^2err(user n,movie m,rating rnm​)=(rnm​−wmT​vn​)2

由于 SGD 效率比较高，所以在大型矩阵分解问题中非常常见。

最后要提到的一点是，算法仅仅是一个工具，了解算法的原理后可以根据自己的需要进行修改。例如随机梯度下降法，如果我们的数据是有时间先后关系的，比如 netflix 的电影推荐系统，用户最近看的电影应该比几年前看的电影更重要，因为这更能体现用户最近的喜好，那么我们就可以修改随机梯度下降法，在最后一百步的时候仅仅从最近的一百个数据中选择数据进行优化参数，这种修改后的方法我们称之为 time-deterministic GD.

矩阵分解的结果也不一定非要直接用来预测，我们可以用分解后得到的中间层的数据作为输入进行神经网络的训练，可能得到比使用完整的矩阵分解进行预测更好的结果。
