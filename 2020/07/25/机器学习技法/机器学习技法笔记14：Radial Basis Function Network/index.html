机器学习技法笔记14：Radial Basis Function Network

Posted on 2020-07-25 In 机器学习技法听课笔记 Valine:

0 说明

径向基函数（Radial Basis Function）是一个取值仅仅依赖于离中心点距离的实值函数，Radial Basis Function Network 是一种把径向基函数和神经网络结合起来的算法，历史上也曾把它看做一种神经网络。

注意，RBF 不是一种常用的方法，一般能够使用 RBF 的地方使用高斯核支持向量机也可以得到相近的结果，这一讲介绍这种方法的目的更多的在于引出聚类方法并且引导大家更加一般地思考高斯核支持向量机方法。

1 RBF Network Hypothesis

前几讲中讲过一个算法叫做 Gaussian SVM，公式是 gSVM(x)=sign(ΣSVαnynexp⁡(−γ∥x−xn∥2)+b)g_{SVM}(x)=sign\left(\Sigma_{SV}\alpha_ny_n\exp(-\gamma\|x-x_n\|^2)+b\right)gSVM​(x)=sign(ΣSV​αn​yn​exp(−γ∥x−xn​∥2)+b) ，几何意义是在一个无限维空间中找一条最宽的线。Gaussion kernel 是 Radial Basis Function (RBF) kernel 的一种，gSVM(X)g_{SVM}(X)gSVM​(X) 可以看作是 Radial Basis Function kernel 的线性聚合，我们将要讲的 RBF Network 也是 Radial Basis Function kernel 的线性聚合。

下图是 RBF 和一般的两层神经网络结构的对比：

可以看到，输出层都是一样的线性聚合，但是第一层却不是线性模型，而是求高斯核表示的一个中心距离。这个模型可以表示为 h(x)=Output(Σm=1MβmRBF(x,μm)+b)h(x)=Output\left(\Sigma_{m=1}^M\beta_mRBF(x,\mu_m)+b\right)h(x)=Output(Σm=1M​βm​RBF(x,μm​)+b) ，高斯核 SVM 是这个函数 的一个特例，表示输出函数是符号函数、径向基函数是高斯核函数的情况。

这个模型的学习是指在给定了径向基函数和输出函数后决定 μm\mu_mμm​ 和 βm\beta_mβm​ 的过程。

我们之前在讲支持向量机和神经网络的时候都说过，它们某种程度上是在衡量相似性，RBF 也是这样，只是它专指径向距离这样的相似性。

2 RBF Network Learning

先看如何决定中心点 μm\mu_mμm​

第一种想法是直接使用所有的点，此时 M=N,μm=xmM=N,\mu_m=x_mM=N,μm​=xm​，这表示每个点对周围都有影响，影响大小取决于 βm\beta_mβm​，若 βm=1⋅ym\beta_m=1\cdot y_mβm​=1⋅ym​ 则表示每个点对周围的影响是接近的。这种使用所有点的方法称之为 full RBF Network.

第二种想法是一个点的函数值仅仅用离它最近的几个点就可以预测，而不需要把所有的点都作为中心，以高斯核函数为例，它的径向距离衰减的很快，比较远的点的数值的投票无法对近距离点的投票产生决定性改变，所以没有必须把他们也作为中心点。这时只需要选择距离预测点最近的一个或几个点作为中心即可。这样也就产生了所谓的 k 近邻（k-Nearest Neighbor, 或 KNN）算法的需要。

另外一种避免过拟合的方法就是不使用所有的中心。为什么这样可以避免过拟合？原因可以参考高斯核的支持向量机，这种模型与前面的 full RBF Network 的区别就在于仅仅使用少量数据点作为中心，这些数据点也就是支持向量。

这个正则化方法的重点就是从大量的数据点中选出一些好的代表当作中心，目前用在这里比较好的算法就是下一节将要介绍的 k 均值聚类算法。

3 k-Means Algorithm

现在的目标是从大量的数据中选出少量好的代表。

选代表的过程大致可以分为两步：把所有数据点分成 m 个互不相交的数据集 S1,…,SMS_1,\ldots,S_MS1​,…,SM​；每个数据集选一个代表 μm\mu_mμm​，这个点不一定是其中的某一个，可以是他们的中心点或其他。

使用平方损失函数的情况下我们的优化目标可以写成：

这是一个组合最优化问题，既要做排列组合的最优化，又要做数值最优化，直接求最优化非常困难，但是如果固定 (S1,…,SM)(S_1,\ldots,S_M)(S1​,…,SM​) 或 (μ1,…,μM)(\mu_1,\ldots,\mu_M)(μ1​,…,μM​) 其中之一对另一个集合最优化则相对简单，所以我们这里采取的方法是对这两个组合交替最优化。

当 (μ1,…,μM)(\mu_1,\ldots,\mu_M)(μ1​,…,μM​) 已经确定时，由于每个 xnx_nxn​ 分到一个集合中，所以很容易看出来使得目标函数最小的方法是把 xnx_nxn​ 分到离它最近的中心点去；当 (S1,…,SM)(S_1,\ldots,S_M)(S1​,…,SM​) 已经确定时，也就是每个点都分到了唯一的一个集合中，这时每个集合中寻找最优代表是可以单独进行的，当找到这些最优代表后把它们合并起来就得到了使得整个目标函数值最小的最优解，所以我们下面单独来看一下对于 S1S_1S1​ 而言怎么求最优代表。

假设 x1,…,xTx_1,\ldots,x_Tx1​,…,xT​ 是 S1S_1S1​ 中包含的所有数据点，现在希望找到一个 μ1\mu_1μ1​ 使得 Σ∥xt−μ1∥2\Sigma\|x_t-\mu_1\|^2Σ∥xt​−μ1​∥2 最小，注意到我们在统计学中有一个结论，使得这样的公式的值最小的 μ1\mu_1μ1​ 应当是 {xt}\{x_t\}{xt​} 的平均值，这一点只需要给括号内加一个再减一个平均值并展开就很容易证明，当数据点是高维数据时，容易计算得到 μ1\mu_1μ1​ 的每个维度的值都是 {xt}\{x_t\}{xt​} 相应的维度的值的平均值。
